# Security & Anti-Spoofing

FacePipe is designed for high-security environments where presentation attacks (spoofing) and synthetic media (deepfakes) are a threat.

## Liveness Detection (Anti-Spoofing)

Presentation attacks occur when someone holds up a phone screen or a printed photo of an authorized user to the camera.

FacePipe integrates lightweight passive liveness checks to prevent this.

### Configuration
```bash
FR_ANTISPOOF_SECURITY_LEVEL=MEDIUM
```
- `LOW`: Only rejects obvious print attacks.
- `MEDIUM`: Rejects most print and screen attacks.
- `HIGH`: Extremely strict. May increase false rejection rate (FRR) for legitimate users in bad lighting.

## Deepfake Detection

With the rise of Generative AI, injection attacks using deepfakes are a growing threat for remote KYC (Know Your Customer) systems.

FacePipe uses a frequency-analysis-based deepfake detector that catches synthetic artifacts generated by GANs and Diffusion models.

```bash
FR_DEEPFAKE_ENABLED=true
FR_DEEPFAKE_THRESHOLD=0.5
```

## Decision Fusion

The real power of FacePipe comes from **Decision Fusion**. Instead of treating recognition and security as separate steps, they are fused. 

If a face has a 99% similarity score to the CEO, but the liveness score is only 40%, the `DecisionFusionEngine` will classify the face as **Ambiguous** or **Spoofed**, and it will *not* be recognized.
